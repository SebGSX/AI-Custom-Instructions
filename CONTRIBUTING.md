# Contributing

## Code of Conduct

All contributors must adhere to the [Code of Conduct](/CODE_OF_CONDUCT.md).

## How to Contribute

### Substance over Form

Contributions are welcome as are corrections. The project follows
[Crocker's Rules](https://www.lesswrong.com/w/crockers-rules). Accordingly, direct, honest, and constructive feedback is
appreciated. Please submit a pull request with your changes or an issue with your feedback.

### Reporting Bugs and Suggesting Enhancements

- Search existing issues to avoid duplicates.
- Use the provided templates for bug reports and enhancement requests.

### Submitting Pull Requests

- Search existing pull requests to avoid duplicates.
- Fork the repository and create a new branch for your changes.
- Use the provided pull request template and ensure that your changes comply with the Pre-Submission Checklist from the
  PR template.

### Style and Conventions

Please follow the existing style and conventions used in the project. These are generally the industry standard, as
operationalized by linters and IDEs. If you identify any inconsistencies, please point them out in
your contribution.

### Use of Artificial Intelligence (AI)

Contributions are permitted to use AI but such use must align with the principle of accelerating high-quality work while
maintaining human review, accountability, and responsibility. AI slop whether submitted by humans or agents will be
rejected and may result in a ban for spam. AI accelerates under supervision; it does not replace engineers.

> "AI slop" includes hallucinated API references, code that doesn't compile, inflated diffs, or descriptions that don't
> accurately reflect the change.

### Commit Guidelines

Please follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) for commit messages.
